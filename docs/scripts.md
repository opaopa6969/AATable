CLI reference for all four AATable scripts.

## aatable.py

Convert Markdown tables, CSV, or TSV to ASCII Art tables with correct CJK column alignment.

### Synopsis

```
python3 aatable.py [OPTIONS] [FILE]
```

### Arguments

| Argument | Description |
|----------|-------------|
| `FILE`   | Input file path. If omitted, reads from stdin. |

### Options

| Option | Short | Values | Default | Description |
|--------|-------|--------|---------|-------------|
| `--format` | `-f` | `auto` `md` `csv` `tsv` | `auto` | Input format. `auto` detects by content: lines starting with `\|` тЖТ Markdown; tab-separated тЖТ TSV; otherwise тЖТ CSV. |
| `--style` | `-s` | `single` `double` `bold` `round` `ascii` | `single` | Box-drawing character set. |
| `--padding` | `-p` | integer | `1` | Spaces added inside each cell on each side. |
| `--no-header` | тАФ | flag | off | Treat the first row as data, not a header. No separator line is drawn after row 0. |
| `--ambiguous-width` | `-a` | `1` `2` | `1` (or profile) | Display width for Unicode Ambiguous characters. `1` for Windows Terminal / VS Code. `2` for macOS Terminal.app. Overrides `~/.aatable_profile.json`. |
| `--demo` | тАФ | flag | off | Print a demo table containing all character types, then print all five styles. Ignores `FILE` and format options. |

### Exit codes

| Code | Meaning |
|------|---------|
| 0    | Success |
| 1    | No valid tabular data found in input |

### Format auto-detection order

1. If any non-empty line starts with `|` тЖТ **Markdown**
2. If any line contains a tab character тЖТ **TSV**
3. Otherwise тЖТ **CSV**

### Box styles

| Name     | Characters used |
|----------|----------------|
| `single` | `тФМ тФА тФм тФР тФВ тФЬ тФ╝ тФд тФФ тФ┤ тФШ` |
| `double` | `тХФ тХР тХж тХЧ тХС тХа тХм тХг тХЪ тХй тХЭ` |
| `bold`   | `тФП тФБ тФ│ тФУ тФГ тФг тХЛ тФл тФЧ тФ╗ тФЫ` |
| `round`  | `тХн тФА тФм тХо тФВ тФЬ тФ╝ тФд тХ░ тФ┤ тХп` |
| `ascii`  | `+ - + + \| + + + + + +` |

### Profile auto-loading

At module load time, `aatable.py` reads `~/.aatable_profile.json` (generated by `aacalibrate.py`) and sets `_ambiguous_width` from the `ambiguous_width` field. The `--ambiguous-width` CLI option overrides the profile.

### Examples

```bash
# Markdown from stdin
echo '| a | b |
|---|---|
| 1 | 2 |' | python3 aatable.py

# CSV file, round style
python3 aatable.py -f csv --style round data.csv

# TSV, no header, bold style
cat data.tsv | python3 aatable.py -f tsv --no-header --style bold

# macOS Terminal, double style
python3 aatable.py -f csv -a 2 --style double data.csv

# psql тЖТ table
psql --csv -c "SELECT * FROM users LIMIT 10" mydb \
  | python3 aatable.py -f csv

# git log тЖТ table
git log --format='%h,%s,%an' -5 \
  | python3 aatable.py -f csv --style round

# demo all styles
python3 aatable.py --demo
```

## mmd2ge.py

Convert a Mermaid flowchart to Graph::Easy input format, inserting zero-width spaces (U+200B) after CJK characters so that Graph::Easy allocates correctly-sized boxes.

### Synopsis

```
python3 mmd2ge.py [FILE]
```

Output is meant to be piped directly to `graph-easy`.

### Arguments

| Argument | Description |
|----------|-------------|
| `FILE`   | Mermaid flowchart file. If omitted (or `-`), reads from stdin. |

### Supported Mermaid syntax

| Feature | Syntax |
|---------|--------|
| Direction | `graph TD` / `graph LR` / `graph RL` / `graph BT` |
| Rectangle node | `A[label]` |
| Rounded rect node | `A(label)` |
| Diamond node | `A{label}` or `A{{label}}` |
| Circle node | `A((label))` |
| Directed edge | `A --> B` |
| Edge with label | `A -- label --> B` or `A -->|label| B` |
| Dotted edge | `A -.-> B` |
| Bold edge | `A ==> B` |
| Undirected edge | `A --- B` |
| Comments | `%% comment` |
| Multi-statement lines | `A --> B; B --> C` |

Unsupported: subgraphs, click handlers, classDef, style directives.

### Output format

Outputs Graph::Easy syntax. A direction declaration is prepended if not `down`:

```
graph { flow: right; }
[ хЕехКЫ\u200b ] --> [ уГСуГ╝уВ╣\u200b ]
[ уГСуГ╝уВ╣\u200b ] -- OK --> [ хЗ║хКЫ\u200b ]
```

### Examples

```bash
# From stdin, pipe to graph-easy
echo 'graph LR
A[Start] --> B[End]' | python3 mmd2ge.py | graph-easy --as=boxart

# From file
python3 mmd2ge.py flow.mmd | graph-easy --as=boxart

# With CJK labels
echo 'graph TD
A[хЕехКЫ] --> B{хИдхоЪ}
B -->|OK| C[хоМф║Ж]
B -->|NG| D[уВиуГйуГ╝]' | python3 mmd2ge.py | graph-easy --as=boxart
```

## aafixwidth.py

Fix CJK width misalignment in ASCII Art text. Reads from stdin or a file, writes corrected text to stdout.

### Synopsis

```
python3 aafixwidth.py [OPTIONS] [FILE]
```

### Arguments

| Argument | Description |
|----------|-------------|
| `FILE`   | Input file. If omitted, reads from stdin. |

### Options

| Option | Short | Values | Default | Description |
|--------|-------|--------|---------|-------------|
| `--ambiguous-width` | `-a` | `1` `2` | `1` | Display width for Unicode Ambiguous characters. Match to your terminal. |

### How it fixes

1. Scans for horizontal border lines (`+----+----+`) to locate column positions
2. For content lines (`| cell | cell |`), measures each cell's `display_width()`
3. Trims trailing spaces: `spaces_to_remove = display_width(content) - len(content)`

If no box structure is detected, falls back to a simpler per-line approach.

### Limitation: find_boxes() dead code

`aafixwidth.py` contains a `find_boxes()` function that traces full rectangular boxes. This function is not called in the current implementation тАФ the active code path uses `find_column_positions()` instead. `find_boxes()` is preserved but unreachable.

### Examples

```bash
# Fix graph-easy output on the fly
graph-easy input.dot | python3 aafixwidth.py

# Fix a saved ASCII Art file
python3 aafixwidth.py broken-aa.txt

# Fix with macOS Ambiguous width
cat broken-aa.txt | python3 aafixwidth.py -a 2
```

## aacalibrate.py

Probe the terminal's actual character rendering widths by writing test characters and querying cursor positions via ANSI DSR (`\033[6n`). Saves a JSON profile that `aatable.py` reads automatically.

### Synopsis

```
python3 aacalibrate.py [OPTIONS]
```

Requires an interactive TTY. Cannot be piped.

### Options

| Option | Short | Values | Default | Description |
|--------|-------|--------|---------|-------------|
| `--output` | `-o` | path | `~/.aatable_profile.json` | Profile output path. |
| `--quick` | `-q` | flag | off | Quick mode: probe only Ambiguous and emoji characters (faster). |
| `--json` | тАФ | flag | off | Write the profile JSON to stdout instead of the output file. |
| `--quiet` | тАФ | flag | off | Suppress all progress output to stderr. |

### Profile format

```json
{
  "terminal": {
    "TERM": "xterm-256color",
    "TERM_PROGRAM": "iTerm.app",
    "WT_SESSION": false,
    "LANG": "ja_JP.UTF-8",
    "WSL_DISTRO_NAME": ""
  },
  "ambiguous_width": 2,
  "probe_results": {
    "ambiguous": [
      { "char": "тСа", "name": "Circled Digit 1", "eaw": "A", "measured_width": 2 },
      ...
    ],
    ...
  }
}
```

The `ambiguous_width` field is the consensus value (mean of all Ambiguous measurements > 1.5 тЖТ 2, else 1).

### Auto-loading by aatable.py

`aatable.py` calls `_load_ambiguous_width_from_profile()` at import time. If `~/.aatable_profile.json` exists and is valid JSON, `ambiguous_width` is read from it. This value is the default; `--ambiguous-width` on the CLI overrides it.

### Examples

```bash
# Full probe, save to default path
python3 aacalibrate.py

# Quick probe (Ambiguous + emoji only)
python3 aacalibrate.py --quick

# Save to a custom path
python3 aacalibrate.py -o ~/my-terminal-profile.json

# Print JSON to stdout (for inspection)
python3 aacalibrate.py --json --quiet
```
